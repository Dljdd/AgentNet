ETHGlobal OpenAgents — Final Project Concept

AgentNet

A decentralized swarm of AI agents that collaboratively build trust scores for on-chain agents — each with an ENS identity, powered by 0G infrastructure, and enforced by a Uniswap v4 hook that makes DeFi safer for everyone.

From Explorer to Living Protocol
The explorer is a window into an agentic system, not the system itself. A swarm of 3 specialized AI agents — with no central coordinator — continuously indexes on-chain agent activity, scores reputation using LLM-powered analysis on 0G Compute, and serves trust scores on-chain via an oracle. Every agent has a persistent ENS identity. A Uniswap v4 hook consumes these scores to gate DeFi access by reputation — making reliable agents cheaper to operate and bad actors expensive.
// The Agent Swarm — 3 core types, no coordinator
🔍 INDEXER AGENT
Crawls on-chain agent registrations, monitors transactions, tracks Uniswap pool interactions. Publishes structured activity logs to 0G Storage. Listens for new agent registrations via 0G DA event stream.
indexer.agentnet.eth
🧠 SCORER AGENT (LLM-POWERED)
The AI core. Runs on 0G Compute — consumes indexed activity, uses an LLM to analyze transaction patterns, and produces both a numeric trust score and a human-readable assessment. Multiple scorer instances can compete; median consensus determines the final score. Writes results to 0G Storage.
scorer-01.agentnet.eth
🔗 ORACLE AGENT
Reads finalized trust scores from 0G Storage and pushes them on-chain to the Trust Oracle contract. The Uniswap v4 hook and explorer UI both query this oracle. Also monitors for stale scores and triggers re-evaluation by broadcasting to scorer agents via 0G DA.
oracle.agentnet.eth
STRETCH GOALS (IF TIME PERMITS)
WATCHDOG Monitors for rug pulls, wash trading, sybil attacks. Broadcasts signed warnings via 0G DA.    ROUTER Helps agents discover trustworthy collaborators for specific tasks (e.g. "find me a reliable price oracle agent").

// ENS — Agent Identity Infrastructure
🪪 EVERY AGENT GETS A NAME UNDER agentnet.eth
Each agent in the network is assigned a subname under agentnet.eth. The ENS record stores the agent's role, trust tier, operator address, 0G Storage key, and capabilities manifest. Discovery, coordination, and the explorer UI all resolve through ENS — not raw addresses.
How ENS Is Used (4 integration points)
AGENT DISCOVERY
Agents find each other by resolving ENS subnames. No hardcoded addresses. oracle.agentnet.eth always points to the current oracle, regardless of redeployment.
METADATA VIA TEXT RECORDS
ENS text records store agent role, trust tier, 0G Storage key, capabilities manifest, and operator contact. The explorer reads all of this from ENS — no separate database needed.
ACCESS GATING IN HOOK
The Uniswap v4 hook checks ENS reverse resolution to confirm the swapping agent has a valid *.agentnet.eth identity before looking up its trust score.
HUMAN-READABLE EXPLORER
The explorer shows scorer-03.agentnet.eth not 0x4a3f.... Users search by name. Trust history is anchored to the ENS identity, not the contract address.
// Uniswap v4 Trust-Gated Hook — Execution Flow
Agent wants to swap → ENS reverse lookup confirms *.agentnet.eth → beforeSwap hook queries oracle → 80+ → reduced fees | 40–79 → standard fees | <40 → higher fees or blocked
Real economic incentive: an agent's ENS-anchored on-chain reputation directly affects its DeFi costs. Behave well, pay less. The hook implements beforeSwap to read from the AgentNet Trust Oracle contract and apply a dynamic fee multiplier.
// 0G Integration — Full Stack Usage
0G STORAGE
Trust records, LLM assessments, agent activity logs, and historical reputation data. The explorer reads directly from 0G Storage — no centralized DB.
0G COMPUTE
Scorer agents run LLM inference on 0G's decentralized compute layer. Trust analysis is verifiably decentralized, not running on a personal server.
0G DA
Agent-to-agent messaging. Indexers announce new data, scorers publish scores, oracle agents broadcast updates. The coordination backbone with no central broker.
// The Explorer — Human Window Into the Protocol
Search by agent ENS name or address → see registration card, full feedback history, trust score with LLM-generated reasoning, network graph of counterparties, reputation-over-time chart, and live activity feed from the swarm. You can see indexer agents crawling, scorer agents publishing, oracle agents pushing to chain. Search scorer-01.agentnet.eth and get its full identity, history, and current trust tier. It's not a static dashboard — it's a live view of a decentralized protocol.
⚡ MCP SERVER — PROGRAMMATIC ACCESS
Other agents and developers can query AgentNet programmatically via an MCP server. Endpoints: getTrustScore(ensName), getAssessment(ensName), getHistory(ensName), getNetworkGraph(ensName). This makes AgentNet composable — any agent can check another agent's reputation before interacting.

// Known Risks & Mitigations
SCOPE CREEP
MVP is exactly 3 agents + 1 hook + 1 explorer + 1 MCP server. Watchdog and Router are stretch goals only. If Day 7 core loop isn't working, all hands on integration — no new features.
COLD START
15-20 seeded agents on testnet by Day 7. Mix of profiles: reliable, erratic, bad actor. Demo shows a new agent entering an already-alive ecosystem.
"WHERE'S THE AI?"
The Scorer agent uses a real LLM on 0G Compute. Trust assessments include natural-language reasoning, not just numbers. This is the demo's "wow moment" — show the written analysis appearing in real-time.
DASHBOARD FATIGUE
Demo opens with the agent swarm in a terminal, not the UI. The explorer is the payoff at minute 2:30, after judges have seen the agents working. Lead with intelligence, end with interface.
HOOK COMPLEXITY
Keep the hook simple: beforeSwap → read oracle → apply fee multiplier. No complex logic in the hook itself. All intelligence lives in the agents. The hook is a consumer, not a producer.
